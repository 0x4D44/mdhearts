# Coverage Improvement Plan
**Goal:** 98% Code Coverage
**Date:** 2026-01-01

## Strategy
We will achieve the target by systematically enabling testing for untested modules and excluding platform-specific UI code that cannot be tested in a headless environment.

## Stage 0: Baseline Correction
**Objective:** Exclude `src/platform/win32.rs` from coverage statistics to establish a realistic baseline.
**Actions:**
1.  Verify if `cargo llvm-cov` supports `--ignore-filename-regex`.
2.  Create/Update a coverage script (e.g., `tools/cov.ps1`) to include this exclusion.
3.  Generate a new report to set the "Adjusted Baseline".

## Stage 1: CLI Integration Testing
**Objective:** Cover `hearts-app/src/cli.rs` and `hearts-app/src/main.rs`.
**Target:** +15% coverage (approx).
**Actions:**
1.  Create `tests/cli_integration.rs` in `hearts-app`.
2.  Use `assert_cmd` or `std::process::Command` to invoke the `mdhearts` binary.
3.  Implement tests for:
    *   Help/Version flags.
    *   Invalid arguments.
    *   `--export-snapshot` (verifying file creation).
    *   `--explain-once` (running a quick AI decision).
    *   `--compare-batch` (short run).
    *   Verify `main()` entry point handling.

## Stage 2: Core & UI Polish
**Objective:** Reach 100% on `hearts-core` and `hearts-ui`.
**Actions:**
1.  `hearts-core`: Add tests for `rank.rs` and `suit.rs` edge cases (FromPrimitive/ToPrimitive).
2.  `hearts-ui`: Add tests for `resource.rs` error paths.

## Stage 3: Bot & Controller Logic
**Objective:** Close gaps in `play.rs`, `search.rs`, and `controller.rs`.
**Actions:**
1.  **Controller:** Test `Controller::new`, state transitions, and timeout logic using a mock platform if possible, or refactor `Controller` to be more testable.
2.  **Bot/Play:** Add tests for specific heuristic branches (e.g., "avoid dump to non-leader", "followup avoids self dump").
3.  **Bot/Search:** Add tests for search limits (time vs nodes), transposition table behavior, and iterative deepening interruptions.

## Stage 4: Final Polish
**Objective:** Address any remaining difficult branches to hit 98%.
**Actions:**
1.  Review updated coverage report.
2.  Target specific lines (often error logging or `unreachable!` paths) that are missed.
3.  Consider adding `#[cfg(not(test))]` or `#[allow(dead_code)]` where appropriate for strictly untestable code if exclusion isn't enough.
