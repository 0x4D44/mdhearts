# Code Coverage Analysis Report
**Date:** 2026-01-01
**Current Total Coverage:** 50.16% (Regions), 48.43% (Lines)

## Executive Summary
The project has a solid foundation of tests for the core game logic (`hearts-core`) and the new UI models (`hearts-ui`), achieving nearly 100% coverage in these areas. However, the application layer (`hearts-app`) significantly drags down the overall metrics due to:
1.  **Untested Platform Code:** `win32.rs` (Native Windows UI) is effectively 0% covered, accounting for ~25% of the codebase.
2.  **Untested CLI Layer:** `cli.rs` and `main.rs` are 0% covered.
3.  **Complex AI Branching:** `bot/search.rs` and `bot/play.rs` have significant missed branches (67-77% coverage).

## Missed Opportunities
1.  **CLI Integration Tests:** The entire `cli.rs` module can be tested via integration tests that invoke the `run` function with varied arguments, mocking the actual game loop or using short-circuit logic.
2.  **Bot Heuristics:** `bot/play.rs` and `bot/search.rs` contain many specific scoring adjustments (weights) that might not be triggered by the current standard test suite.
3.  **Telemetry:** `telemetry.rs` is easily testable but currently under-covered.

## Strategy for 98% Target
To reach 98% coverage:
1.  **Exclusions:** We must rigorously define "testable code". `win32.rs` should likely be excluded from the coverage denominator unless we implement a comprehensive Win32 mock, which is out of scope for standard unit testing.
2.  **CLI Testing:** Implement a suite of integration tests for `cli.rs`.
3.  **Logic Deep Dive:** Write targeted unit tests for the missed branches in `bot/play.rs` and `bot/search.rs`.
