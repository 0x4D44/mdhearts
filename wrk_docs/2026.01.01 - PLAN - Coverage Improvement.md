# Coverage Improvement Plan
**Goal:** Improve Code Coverage to 98% (of testable logic).

## Strategy
We will tackle the codebase in layers, starting with the easiest gains (CLI, Utilities) and moving to the complex logic (Bot, Controller). We will also formalize the exclusion of the `win32` platform layer, as it requires a UI environment to test.

## Stages

### Stage 1: Infrastructure & Utilities (Target: ~70%)
*   **Module:** `hearts-app/src/telemetry.rs`
    *   **Action:** Add unit tests for telemetry sinking, rotation, and file I/O errors (mocked).
*   **Module:** `hearts-app/src/endgame_export.rs`
    *   **Action:** Add tests for JSON serialization and file writing of endgame states.
*   **Module:** `hearts-app/src/dataset.rs`
    *   **Action:** Cover the remaining dataset recording logic.

### Stage 2: CLI Integration (Target: ~80%)
*   **Module:** `hearts-app/src/cli.rs`
    *   **Action:** Refactor `cli.rs` to allow dependency injection or "dry run" modes for commands.
    *   **Action:** Create a new test suite `tests/cli_integration.rs` that calls `cli::run` with various flag combinations (`--help`, `--version`, `--show-weights`, invalid args).

### Stage 3: Bot Logic Deep Dive (Target: ~90%)
*   **Module:** `hearts-app/src/bot/play.rs` (Normal AI)
    *   **Action:** Analyze missed branches. Likely specific scoring conditions (e.g., "Moon Blocking" logic, "Safe Queen Dump" logic).
    *   **Action:** Create "scenario tests" that set up specific hands and board states to trigger these edge cases.
*   **Module:** `hearts-app/src/bot/search.rs` (Hard AI)
    *   **Action:** Cover the "Search" and "Continuation" logic branches.
    *   **Action:** Test the `Hard` bot's configuration overrides (timeouts, step limits).

### Stage 4: Controller & Orchestration (Target: ~95%)
*   **Module:** `hearts-app/src/controller.rs`
    *   **Action:** Test the threading model and timeouts. Use a `MockBot` to simulate slow/crashing AI.
    *   **Action:** Test state transitions (Passing -> Playing -> Scoring).

### Stage 5: Platform Exclusion & Final Polish (Target: 98%)
*   **Module:** `hearts-app/src/platform/win32.rs`
    *   **Action:** Apply `#[cfg(not(test))]` or configure `llvm-cov` to exclude this file from the denominator. It is technically "unsafe FFI" code.
*   **Action:** Run a final sweep to catch any remaining uncovered lines in `hearts-core` or `hearts-ui`.

## Execution Order
1.  Configure exclusion for `win32.rs` immediately to see the "true" logical coverage.
2.  Implement Stage 1 (Utilities).
3.  Implement Stage 2 (CLI).
4.  Implement Stage 3 (Bot).
5.  Implement Stage 4 (Controller).
